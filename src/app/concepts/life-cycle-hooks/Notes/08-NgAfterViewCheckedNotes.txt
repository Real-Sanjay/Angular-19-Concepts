ngAfterViewChecked Lifecycle Hook in Angular
Definition
ngAfterViewChecked() is a lifecycle hook that Angular calls after it checks the component’s view and its child views.
It runs every time the change detection cycle is triggered after the view has been initialized.

When Is It Called?
Hook	                Called When?
ngAfterViewInit()	    Once, after view is first initialized
ngAfterViewChecked()	After every change detection cycle, once view is checked

Execution Order (Simplified)
ngOnInit
ngDoCheck
ngAfterContentInit
ngAfterContentChecked
ngAfterViewInit

ngAfterViewChecked ← (Fires on every change detection after view is initialized)

Example Scenario (Recap + Expansion)
AppComponent:
Generates a random number on button click
Uses ngDoCheck to log detection cycles

ts
export class AppComponent {
  randomNumber = Math.random();

  generateRandom() {
    this.randomNumber = Math.random();
  }

  ngDoCheck() {
    console.log('AppComponent - DoCheck');
  }
}
AppComponent Template:
html
<button (click)="generateRandom()">Generate Random Number</button>
<app-test>
  {{ randomNumber }}
</app-test>

TestComponent Template:
html
<div #wrapper>
  <ng-content></ng-content>
</div>

TestComponent TS:
ts
import {
  AfterViewChecked,
  AfterViewInit,
  Component,
  ElementRef,
  ViewChild
} from '@angular/core';

@Component({
  selector: 'app-test',
  templateUrl: './test.component.html'
})
export class TestComponent implements AfterViewInit, AfterViewChecked {
  @ViewChild('wrapper') wrapper!: ElementRef;

  ngAfterViewInit(): void {
    console.log('TestComponent - AfterViewInit');
  }

  ngAfterViewChecked(): void {
    console.log('TestComponent - AfterViewChecked');
  }
}

Use Cases
DOM read/update operations after every view check
Detecting DOM changes not related to Angular binding
Complex condition checks after view rendering

⚠️ Caution
Don’t manipulate DOM or update component state directly here.
That might cause infinite loops or ExpressionChanged errors.
If needed, wrap it in a setTimeout() or use ChangeDetectorRef cautiously.

Interview Q&A Summary
Question	                                    Answer
Q1: What is ngAfterViewChecked() used for?	    It’s a hook triggered after Angular checks the view and child views during each change detection cycle.
Q2: How is it different from ngAfterViewInit()?	ngAfterViewInit() runs only once. ngAfterViewChecked() runs every time the view is checked.
Q3: Can I modify data in ngAfterViewChecked()?	Not directly — doing so might lead to Angular errors. Use setTimeout() or ChangeDetectorRef.
Q4: When does Angular call this hook?	        After every change detection cycle, post view check.
