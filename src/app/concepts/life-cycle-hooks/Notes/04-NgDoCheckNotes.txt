What is ngDoCheck in Angular?
ngDoCheck is a lifecycle hook that gets called during every change detection cycle.
It is used to implement custom change detection logic that Angular‚Äôs default mechanism may not catch.
Unlike ngOnChanges, which only detects changes in @Input() properties, ngDoCheck runs regardless of input changes, giving more manual control.


Key Differences: ngOnChanges vs ngDoCheck
Feature	    ngOnChanges	                                                ngDoCheck
Triggered	Only on changes to @Input() properties	                    Every change detection cycle
Parameters	Receives SimpleChanges with current/previous values	        No parameters
Use Case	Track input changes easily	                                Custom detection (deep objects/arrays)
Efficiency	Lightweight	                                                Can be performance-intensive if misused
Complexity	Simple and declarative	                                    Manual comparison logic needed
Can Be Used Together?	‚ùå Not recommended ‚Äì can lead to complex behavior and bugs	

üí° When to Use ngDoCheck
Use it when:
You need to detect deep changes in nested objects or arrays.
External libraries mutate component state outside of Angular‚Äôs detection.
You are manually managing component state that isn't passed as @Input.

üß™ Example Scenario: Object Property Change Not Detected by ngOnChanges
code:
@Input() user: { name: string };

ngOnChanges(changes: SimpleChanges) {
  console.log("ngOnChanges", changes);
}

// Won't detect when only `user.name` changes without changing the reference
Use ngDoCheck instead:

code:
ngDoCheck() {
  if (this.user.name !== this.previousUserName) {
    console.log("User name changed");
    this.previousUserName = this.user.name;
  }
}

üìö How to Detect Array Changes Using ngDoCheck
code:
items = [1, 2, 3];
previousItems = [...this.items];

ngDoCheck() {
  if (this.items.length !== this.previousItems.length) {
    console.log("Array changed");
    this.previousItems = [...this.items];
  }
}

‚ùì Can ngDoCheck Replace ngOnChanges?
Not entirely. While it can be used for more complex detection, ngOnChanges is:
Easier to use
Provides SimpleChanges object
More efficient when you just want to track @Input() changes
Use ngDoCheck only when necessary.
