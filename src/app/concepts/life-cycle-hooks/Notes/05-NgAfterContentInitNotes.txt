What is ngAfterContentInit in Angular?
Definition:
ngAfterContentInit is a lifecycle hook in Angular that is called once after Angular projects external content into the component using <ng-content>.

When does it run?
It runs after the first ngDoCheck().
It is called only once, even if projected content changes later.

üîÑ Lifecycle Order (Simplified):
ngOnInit() ‚Üí after component‚Äôs input properties initialized.

ngDoCheck() ‚Üí runs on every change detection.

ngAfterContentInit() ‚Üí after content (via <ng-content>) has been projected.


üî¨ When to Use ngAfterContentInit?
Use it when:
You need to interact with projected content (from parent to child).
You are using @ContentChild() to access a projected DOM or component.
You want to run logic after the DOM inside <ng-content> is available.

‚ùìWhat Happens If Projected Content Changes Later?
ngAfterContentInit() is NOT called again.
Use ngDoCheck() or Angular‚Äôs reactive patterns (@Input(), BehaviorSubject, etc.) to detect updates.

Interview Style Questions and Answers
Q1: What is ngAfterContentInit?
It is a lifecycle hook that is called once after Angular has fully initialized all content projected into a component using <ng-content>.

Q2: When would you use ngAfterContentInit instead of ngOnInit?
When your logic depends on content projected from a parent using <ng-content>. Unlike ngOnInit, this hook guarantees that the projected content is fully available.

Q3: Does ngAfterContentInit run when projected content changes?
No. It runs only once. To handle updates, use ngDoCheck() or reactive patterns.
